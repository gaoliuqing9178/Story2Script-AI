# Sprint Contract

Feature ID: P2-LLM-001

Owner: generator agent

Date: 2026-06-06

## 本轮目标

实现 OpenAI-compatible provider 的单阶段小说转 YAML 生成链路：

- `LLM_PROVIDER=openai` 时，`OpenAIProvider` 调用 `OPENAI_BASE_URL/responses`。
- 请求读取 `OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_MODEL`，不读取或提交真实 `.env`。
- `POST /api/screenplay/generate` 接收小说文本，一次性构造 novel-to-YAML prompt。
- provider 返回的 YAML 立即进入现有 `validateScreenplayYamlStructure`，并返回 `{ yaml, validation }`。
- 保留默认 `MockProvider` 路径，mock 模式继续零配置可用。

## 明确不做

- 不实现 Phase 3 多阶段 pipeline。
- 不实现 YAML repair。
- 不引入真实密钥、`.env`、token 或私有模型配置。
- 不把真实外部 OpenAI API 可用性作为本轮自动化验收前提。
- 不重做前端 UI 结构或文案，只让已有小说输入文本传给 generate API。

## 用户路径

1. 本地环境设置 `LLM_PROVIDER=openai`、`OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_MODEL`。
2. 用户或前端调用 `POST /api/screenplay/generate`，请求体传入 `novel`、`novel_text` 或 `text`。
3. 系统构造单阶段 prompt，要求模型只返回符合 v1.0 Schema 的 YAML。
4. `OpenAIProvider` 调用 OpenAI-compatible Responses API。
5. 系统剥离可能出现的 YAML Markdown code fence。
6. 系统立即执行现有 YAML 结构校验与应用层引用校验。
7. 响应返回 `{ yaml, validation }`，validation 可为 `valid:true` 或包含精确错误路径。
8. 切回默认 mock 后，空请求或 UI 请求仍返回 `examples/screenplay-sample.yaml` 并通过校验。

## 数据状态

- 输入：小说全文字符串，推荐使用 `examples/novel-sample.md`。
- 输出：模型返回的 YAML 字符串与 `ValidationResult`。
- 持久化：无。
- 是否影响 `examples/*`：不修改示例，只读取 `examples/novel-sample.md` 与 `examples/screenplay-sample.yaml` 作为测试 fixture。

## UI 要求

- 页面区域：不新增 UI。
- Loading：沿用现有生成按钮 loading。
- 错误态：沿用现有错误展示；openai 模式缺小说文本由后端返回 HTTP 400。
- 空态：不改。

## API 要求

- Endpoint：`POST /api/screenplay/generate`
- Request：

```json
{
  "novel": "第一章...",
  "adaptation_type": "screenplay",
  "adaptation_intensity": "balanced"
}
```

`novel`、`novel_text`、`text` 三者任一非空即可。`adaptation_type` 非法或缺失时默认 `screenplay`。

- Response：

```json
{
  "yaml": "schema_version: \"1.0\"...",
  "validation": {
    "valid": true,
    "errors": [],
    "warnings": []
  }
}
```

- Error：
  - `400 BAD_REQUEST`：openai 模式下缺少小说文本。
  - `502 LLM_UNAVAILABLE`：provider 配置缺失、网络失败、非 JSON 响应、响应缺少 output text。

## 验收方式

```powershell
pnpm verify
pnpm test:ui
```

补充真实路径验证：

1. `apps/server/tests/screenplay-route.test.ts` 启动真实 Express app。
2. 测试启动本地 OpenAI-compatible `/responses` HTTP 服务，不依赖外网或真实 key。
3. 使用 `examples/novel-sample.md` 作为请求输入。
4. 验证 provider 请求包含模型、Authorization、`instructions` 和 `input`。
5. 验证 YAML code fence 会被剥离，返回 YAML 立即通过 validator。
6. 验证 provider 返回坏 YAML 时，response 仍为 200，`validation.errors` 给出 `project.title` 精确错误。
7. 验证 openai 模式缺小说文本返回 400，缺 key 返回 502。
8. 验证 mock 模式仍返回合法 fixture 且 `validation.valid:true`。

## 失败阈值

- 任一验证命令失败：不允许 `passes:true`。
- openai 模式未调用 OpenAI-compatible `/responses`：不允许 `passes:true`。
- provider 结果未立即进入 validator：不允许 `passes:true`。
- mock 默认路径回归：不允许 `passes:true`。
- 状态文件未更新：不允许收尾。
