# Sprint Contract: P0-E2E-001

Feature ID: P0-E2E-001

Owner: generator agent

Date: 2026-06-05

## 本轮目标

正式验收 Phase 0 mock 端到端链路：`MockProvider` 返回仓库内写死的合法剧本 YAML，前端通过后端 `/api/screenplay/generate` 获取并展示该 YAML。

## 明确不做

- 不实现 Phase 1 的真实 YAML 校验器。
- 不接入 OpenAI 或其他真实 LLM。
- 不实现切章、编辑器、预览、导出或修复链路。
- 不改弱 `feature_list.json` 中的验收步骤。

## 用户路径

1. 用户打开 `http://127.0.0.1:5173`。
2. 用户点击“用样例生成”按钮。
3. 前端请求 `POST /api/screenplay/generate`。
4. 后端默认使用 `MockProvider` 读取 `examples/screenplay-sample.yaml`。
5. 页面 `YAML 输出` 区展示包含 `schema_version`、`project`、`source`、`characters`、`locations`、`scenes` 的 YAML。

## 数据状态

- 输入：Phase 0 暂不消费小说文本，按钮请求体为 `{}`。
- 输出：`yaml` 字符串和 Phase 1 前的 mock `validation` 响应形状。
- 持久化：无数据库或浏览器持久化。
- 是否影响 `examples/*`：不修改示例文件；`examples/screenplay-sample.yaml` 是本轮 mock fixture。

## UI 要求

- 页面区域：保留小说输入占位区和 YAML 输出区。
- Loading：点击后按钮进入 `生成中...`，并禁用重复提交。
- 错误态：请求失败时在 YAML 输出区上方显示具体错误。
- 空态：生成前显示“点击‘用样例生成’后，这里会显示后端 mock YAML。”

## API 要求

- Endpoint：`POST /api/screenplay/generate`
- Request：`{}`
- Response：`{ yaml: string, validation: ValidationResult }`
- Error：服务端异常走统一 `{ error: { code, message } }`

说明：`validation` 在 Phase 1 前只提供响应形状，不作为真实校验结果。P0-E2E-001 只验收 mock YAML 经后端 API 到达前端并完整显示。

## 验收方式

```powershell
pnpm verify
pnpm test:ui
```

补充真实路径验证：

1. server Vitest 调用真实 Express app，确认 API 返回 YAML 可解析，并包含六个必需顶层字段。
2. Playwright Chromium 打开首页，点击“用样例生成”，确认页面显示六个必需顶层字段。

## 失败阈值

- 任一验证命令失败：不允许 `passes:true`。
- 页面无法通过后端 API 显示 YAML：不允许 `passes:true`。
- YAML 缺少任一必需顶层字段：不允许 `passes:true`。
- 状态文件未更新：不允许收尾。
